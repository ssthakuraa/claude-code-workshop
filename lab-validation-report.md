### ISSUE-04 — WARN | Lab 10 SHOW GRANTS command uses wrong host

**Lab affected:** Lab 10 (MySQL MCP)
**File:** `labs/lab-10-mcp-mysql.md`
**Symptom:** The verification step reads:

```bash
mysql -h 127.0.0.1 -u root -proot123 -e "SHOW GRANTS FOR 'hr_readonly'@'%';"
```

But `create-readonly-user.sql` creates the user as `'hr_readonly'@'localhost'`, not `'hr_readonly'@'%'`. The SHOW GRANTS command returns empty/error, confusing students into thinking setup failed.

**Fix:** Updated lab-10 to use `'hr_readonly'@'localhost'` in the SHOW GRANTS command.
**Also fixed:** `create-readonly-user.sql` updated to grant both `@'localhost'` and `@'127.0.0.1'` — the MCP server connects via `127.0.0.1` (TCP) not the Unix socket, so `@'localhost'`-only grants can silently fail.
